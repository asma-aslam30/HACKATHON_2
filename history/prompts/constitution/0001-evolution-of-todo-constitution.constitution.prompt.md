## Prompt

Create a global project Constitution for my Hackathon II project called "Evolution of Todo – Spec‑Driven, AI‑Native Todo System".

This Constitution must cover all 5 phases described in the official hackathon document:

Phase I: In‑Memory Python Console App

Phase II: Full‑Stack Web Application (Next.js, FastAPI, SQLModel, Neon, Better Auth)

Phase III: AI‑Powered Todo Chatbot (OpenAI ChatKit, Agents SDK, MCP)

Phase IV: Local Kubernetes Deployment (Docker, Minikube, Helm, AIOps tools)

Phase V: Advanced Cloud Deployment (Kubernetes + Kafka + Dapr + CI/CD)

Write the Constitution with these sections:

Purpose and Vision

Explain that this project evolves a simple Todo CLI into a full cloud‑native, AI‑powered, event‑driven Todo platform across five phases.

Core Principles
Include at least these principles:

Spec‑First Development: Every feature, API, model, UI, and infrastructure change must start as a written spec in /specs.

AI‑Implemented Only: All application code is generated or edited by AI according to specs; the human does not hand‑write code.

Single Source of Truth: The Constitution and specs govern behavior; if code and runtime behavior disagree, specs must be fixed first.

Evolutionary Architecture: Earlier phases are incrementally evolved into later phases, not thrown away.

Scope of the Project

Briefly summarize what must be delivered in each of the 5 phases, and mention that Basic, Intermediate, and Advanced Todo features (CRUD, priorities/tags, search/filter/sort, recurring tasks, reminders) will be implemented across phases.

Agents and Responsibilities

Define the following agents, each in 2–3 sentences, describing responsibility and ownership:

Project Manager Agent

System Architect Agent

Spec Architect Agent

Backend / FastAPI Pro Agent

Frontend UI/UX Agent

Console App Agent

AI Chatbot Agent

CloudOps & Kubernetes Agent

Testing & QA Agent

Code Quality & Integration Agent

Skills and Workflows

List broad skill groups (no need to list tiny sub‑skills):

Specification skills

Coding & quality skills

UX & experience skills

Backend & security skills

AI & language skills

Cloud & operations skills

Product & analytics skills

Describe the standard workflow that all work must follow:

Plan → Write/Update Specs → Implement with AI → Test → Review & Refine.

Constraints and Non‑Negotiables

State clearly that:

Manual coding by the human is not allowed; only AI writes/edits code.

Every feature must be testable and documented.

All major technical decisions (schemas, services, Kafka topics, Dapr components, Kubernetes changes) must be captured in specs or short architecture notes.

Success Criteria

Define success as:

All 5 phases implemented as described in the hackathon doc, including the AI chatbot and cloud‑native deployment.

Repository is self‑explanatory to another engineer or AI using only the Constitution and the specs.

Reusable intelligence is visible through documented agents, skills, and architectural patterns that can be reused in other projects.

Formatting & tone:

Output valid Markdown suitable for a file named Constitution.md.

Keep the tone clear, professional, and easy for students to understand.

This Constitution is project‑wide; do not dive into very detailed specs of any single phase.
