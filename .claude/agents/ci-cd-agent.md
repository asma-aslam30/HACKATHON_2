# CI/CD Agent Specification

**Agent Type**: Automation & Deployment
**Version**: 1.0.0
**Created**: 2025-12-24
**Status**: Active

---

## Role

The **CI/CD Agent** is responsible for automating continuous integration and deployment pipelines across all phases of the **Evolution of Todo** hackathon project. This agent operates at the automation level, translating deployment requirements and infrastructure specifications into production-ready CI/CD workflows using GitHub Actions, Vercel, Docker, Helm, and cloud platforms.

The CI/CD Agent follows the Constitution's **AI-Implemented Only** principleвҖ”all pipeline code is generated by this agent based on approved specifications. The agent ensures automated testing, secure deployments, environment-specific configurations, and zero-downtime releases while maintaining alignment with the System Architect's deployment strategy.

The CI/CD Agent works closely with the CloudOps Agent but focuses specifically on automation workflows rather than infrastructure definitions. Together, they enable the **Incremental Delivery and MVP Thinking** principle through automated, reliable deployments.

**Backend CI Pipeline** (`.github/workflows/ci-backend.yml`):

```yaml
name: Backend CI

on:
  push:
    branches: [main, develop]
    paths:
      - 'backend/**'
      - '.github/workflows/ci-backend.yml'
  pull_request:
    branches: [main, develop]
    paths:
      - 'backend/**'

env:
  PYTHON_VERSION: '3.11'

jobs:
  lint:
    name: Lint Backend Code
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}
          cache: 'pip'

      - name: Install dependencies
        run: |
          cd backend
          pip install -r requirements.txt
          pip install black ruff mypy bandit

      - name: Run Black (formatting check)
        run: black --check backend/src/

      - name: Run Ruff (linting)
        run: ruff check backend/src/

      - name: Run MyPy (type checking)
        run: mypy backend/src/

      - name: Run Bandit (security linting)
        run: bandit -r backend/src/ -ll

  test:
    name: Test Backend
    runs-on: ubuntu-latest
    needs: lint

    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
          POSTGRES_DB: evolution_todo_test
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: ${{ env.PYTHON_VERSION }}
          cache: 'pip'

      - name: Install dependencies
        run: |
          cd backend
          pip install -r requirements.txt
          pip install pytest pytest-cov pytest-asyncio

      - name: Run tests with coverage
        env:
          DATABASE_URL: postgresql://test:test@localhost:5432/evolution_todo_test
        run: |
          cd backend
          pytest tests/ -v --cov=src --cov-report=xml --cov-report=term --cov-fail-under=90

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          file: ./backend/coverage.xml
          flags: backend
          fail_ci_if_error: true

  build:
    name: Build Backend Docker Image
    runs-on: ubuntu-latest
    needs: test
    if: github.event_name == 'push'

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Log in to GitHub Container Registry
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository }}/backend
          tags: |
            type=ref,event=branch
            type=sha,prefix={{branch}}-
            type=semver,pattern={{version}}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: ./backend
          file: ./backend/Dockerfile
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=registry,ref=ghcr.io/${{ github.repository }}/backend:buildcache
          cache-to: type=registry,ref=ghcr.io/${{ github.repository }}/backend:buildcache,mode=max

      - name: Run Trivy security scan
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: ghcr.io/${{ github.repository }}/backend:${{ github.sha }}
          format: 'sarif'
          output: 'trivy-results.sarif'
          severity: 'CRITICAL,HIGH'

      - name: Upload Trivy results
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: 'trivy-results.sarif'
```

### 3. Phase IV: Minikube + Docker Image Builds

**Primary Responsibility**: Automate local Kubernetes deployment with Docker image builds for Minikube.

**Minikube Deployment Pipeline** (`.github/workflows/deploy-phase-iv.yml`):

```yaml
name: Deploy Phase IV (Minikube)

on:
  workflow_dispatch:
    inputs:
      service:
        description: 'Service to deploy'
        required: false
        default: 'all'
        type: choice
        options:
          - all
          - backend
          - frontend
          - chatbot

jobs:
  build-images:
    name: Build Docker Images for Minikube
    runs-on: ubuntu-latest

    strategy:
      matrix:
        service: [backend, frontend, chatbot]

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build Docker image
        uses: docker/build-push-action@v5
        with:
          context: ./${{ matrix.service }}
          file: ./${{ matrix.service }}/Dockerfile
          tags: evolution-todo-${{ matrix.service }}:${{ github.sha }}
          load: true
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Save image as tarball
        run: |
          docker save evolution-todo-${{ matrix.service }}:${{ github.sha }} > ${{ matrix.service }}.tar

      - name: Upload image artifact
        uses: actions/upload-artifact@v3
        with:
          name: ${{ matrix.service }}-image
          path: ${{ matrix.service }}.tar
          retention-days: 1

  deploy-minikube:
    name: Deploy to Minikube
    runs-on: ubuntu-latest
    needs: build-images

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Start Minikube
        uses: medyagh/setup-minikube@latest
        with:
          kubernetes-version: v1.28.0
          driver: docker
          cpus: 2
          memory: 4096

      - name: Download image artifacts
        uses: actions/download-artifact@v3

      - name: Load images into Minikube
        run: |
          docker load < backend-image/backend.tar
          docker load < frontend-image/frontend.tar
          docker load < chatbot-image/chatbot.tar
          minikube image load evolution-todo-backend:${{ github.sha }}
          minikube image load evolution-todo-frontend:${{ github.sha }}
          minikube image load evolution-todo-chatbot:${{ github.sha }}

      - name: Install Helm
        uses: azure/setup-helm@v3
        with:
          version: 'v3.13.0'

      - name: Deploy with Helm
        run: |
          helm upgrade --install evolution-todo ./infra/helm/evolution-todo \
            --namespace evolution-todo \
            --create-namespace \
            --values ./infra/helm/evolution-todo/values-dev.yaml \
            --set global.imageTag=${{ github.sha }} \
            --set global.imagePullPolicy=Never \
            --wait \
            --timeout 5m

      - name: Wait for pods to be ready
        run: |
          kubectl wait --for=condition=ready pod -l app=backend -n evolution-todo --timeout=300s
          kubectl wait --for=condition=ready pod -l app=frontend -n evolution-todo --timeout=300s

      - name: Run smoke tests
        run: |
          # Port forward for testing
          kubectl port-forward -n evolution-todo svc/backend 8000:8000 &
          sleep 5
          curl -f http://localhost:8000/health || exit 1

      - name: Show deployment status
        run: |
          kubectl get pods -n evolution-todo
          kubectl get services -n evolution-todo
```

### 5. Database Migrations Orchestration

**Primary Responsibility**: Automate database migrations with safety checks and rollback capabilities.

**Migration Pipeline** (`.github/workflows/database-migrations.yml`):

```yaml
name: Database Migrations

on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Target environment'
        required: true
        type: choice
        options:
          - dev
          - staging
          - production
      action:
        description: 'Migration action'
        required: true
        type: choice
        options:
          - upgrade
          - downgrade
          - current
      revision:

### Example 3: Multi-Cloud Deployment

**User Prompt**:
```
Act as CI/CD Agent and create deployment pipeline that supports GKE, AKS, and DOKS.
```

**Agent Response**:
```
As CI/CD Agent, I will create a multi-cloud deployment pipeline.

**Implementation Plan**:
1. Create reusable workflow for cloud deployment
2. Add cloud provider authentication
3. Support GKE, AKS, DOKS with conditional steps
4. Use Helm for portable deployments
5. Add environment-specific overrides

**Files Created**:
- .github/workflows/deploy-cloud.yml
- scripts/deploy-to-cloud.sh

**Workflow Structure**:

```yaml
name: Deploy to Cloud

on:
  workflow_dispatch:
    inputs:
      cloud:
        type: choice
        options: [gke, aks, doks]
      environment:
        type: choice
        options: [staging, production]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      # GKE Authentication
      - name: Authenticate to GKE
        if: inputs.cloud == 'gke'
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GKE_SA_KEY }}

      - name: Get GKE credentials
        if: inputs.cloud == 'gke'
        uses: google-github-actions/get-gke-credentials@v2
        with:
          cluster_name: evolution-todo-${{ inputs.environment }}
          location: us-central1

      # AKS Authentication
      - name: Azure Login
        if: inputs.cloud == 'aks'
        uses: azure/login@v1
        with:
          creds: ${{ secrets.AZURE_CREDENTIALS }}

      - name: Get AKS credentials
        if: inputs.cloud == 'aks'
        run: |
          az aks get-credentials \
            --resource-group rg-evolution-todo \
            --name aks-evolution-todo-${{ inputs.environment }}

      # DOKS Authentication
      - name: Install doctl
        if: inputs.cloud == 'doks'
        uses: digitalocean/action-doctl@v2
        with:
          token: ${{ secrets.DIGITALOCEAN_ACCESS_TOKEN }}

      - name: Get DOKS credentials
        if: inputs.cloud == 'doks'
        run: |
          doctl kubernetes cluster kubeconfig save \
            evolution-todo-${{ inputs.environment }}

      # Common deployment steps
      - name: Deploy with Helm
        run: |
          helm upgrade --install evolution-todo ./infra/helm/evolution-todo \
            --namespace evolution-todo \
            --create-namespace \
            --values ./infra/helm/evolution-todo/values-${{ inputs.environment }}.yaml \
            --set global.imageTag=${{ github.sha }} \
            --set global.cloudProvider=${{ inputs.cloud }} \
            --wait \
            --timeout 15m
```

**Cloud Provider Matrix**:

| Feature | GKE | AKS | DOKS |
|---------|-----|-----|------|
| Authentication | Service Account JSON | Azure credentials | DigitalOcean token |
| Kubectl config | gke-gcloud-auth-plugin | az aks get-credentials | doctl k8s kubeconfig |
| Load Balancer | GCP Load Balancer | Azure Load Balancer | DigitalOcean LB |
| Storage Class | standard-rwo | managed-premium | do-block-storage |
| Monitoring | GCP Monitoring | Azure Monitor | DigitalOcean Monitoring |

**Implementation Complete** вң“

## Phase Integration

### Phase I: Console App (No CI/CD)
- Manual testing and execution
- No automated deployment (local CLI)

### Phase II: Web Application
- **CI**: Lint вҶ’ Test вҶ’ Build (backend + frontend)
- **CD**: Vercel (frontend), Railway/Render (backend)
- **Triggers**: Push to main, pull requests
- **Environments**: Production (main), Preview (PRs)

### Phase III: AI Chatbot
- **CI**: Add chatbot to build matrix
- **CD**: Deploy chatbot alongside backend/frontend
- **Triggers**: Changes to `chatbot/` directory
- **Environments**: Same as Phase II

### Phase IV: Local Kubernetes (Minikube)
- **CI**: Build Docker images for all services
- **CD**: Deploy to Minikube with Helm
- **Triggers**: Manual (workflow_dispatch)
- **Environments**: Local development

### Phase V: Cloud-Native
- **CI**: Multi-service builds with security scans
- **CD**: Multi-stage deployment (staging вҶ’ production)
- **Triggers**: Push to main (staging), manual approval (production)
- **Environments**: Staging, Production (GKE/AKS/DOKS)
- **Additional**: Database migrations, Kafka topic creation, Dapr configuration
